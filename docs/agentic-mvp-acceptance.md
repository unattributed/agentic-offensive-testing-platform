# Agentic MVP Acceptance

The agentic MVP is accepted when AOTP can run a multi-iteration LangChain Deep Agent campaign with
a local Ollama model against an authorized target and:

- execute real campaign-governed FOSS tools through typed native wrappers;
- deny malformed, out-of-scope, over-budget, or unapproved calls;
- write hashed and classified evidence;
- preserve sensitive proof material in an encrypted campaign vault;
- allow authorized agent and tool access to raw vault contents under active ROE and log each use;
- adapt the next test from classified evidence and coverage;
- stop, steer, resume, and replay without bypassing current policy;
- produce an evidence-linked draft report with an optional approval-gated sensitive annex; and
- keep submission and disclosure manual-only.

Acceptance also requires a local-only Ollama endpoint, no MCP dependency, no paid-tool requirement,
repository-safety validation, negative policy tests, and an operator demo using an explicitly
authorized target or synthetic fixture.
