# Demonstration examples

These files are reproducible placeholder outputs, not assessment evidence.

`dry-run-summary.example.json` is the stable summary produced by
`scripts/run-evaluator-demo.sh`. Generated state, events, manifests, and reports remain ignored
under the selected workspace's `.aotp/` directory.

The summary intentionally excludes timestamps, paths, run identifiers, event hashes, and manifest
hashes. It records only the safety-relevant demonstration result: placeholder inputs, zero
requests, completed dry-run objectives, evidence count, report-ready finding count, and report
limitation status.
