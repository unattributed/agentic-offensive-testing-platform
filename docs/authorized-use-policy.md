# Authorized use policy

AOTP may be used only when the operator has written authority for the exact assets, accounts, services, APIs, test windows, and test categories involved. Scope is authoritative.

General consent, safe harbor, evaluator access, or a confidentiality agreement does not authorize an omitted asset or action. The operator must stop on policy denial, instability, lockout risk, redaction failure, an emergency request, or ambiguity that requires human review.

Automatic enrollment, unsolicited discovery, target expansion, destructive testing, credential attacks, automatic submission, and public disclosure are outside the platform model.
