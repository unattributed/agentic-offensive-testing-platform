# Live assessment scope model

A live scope must contain:

- non-placeholder authorization, agreement, program-profile, rules-of-engagement, and applicable confidentiality references;
- explicit target aliases, domains, services, APIs, provisioned environments, and accounts;
- permitted windows and testing categories;
- fuzzing, panel, SBOM, configuration, and cryptographic permissions;
- forbidden actions and quantitative limits;
- evidence handling, reporting, disclosure, and emergency-stop rules; and
- operator approval.

Aliases appear in evidence. Raw sensitive values remain in the private scope. Missing or malformed fields deny execution.
